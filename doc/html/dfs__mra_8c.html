Default data distribution algorithm.

lista de workers -> workers_hosts[id] (array); pegar capacidade -> MSG_get_host_speed (config_mra.workers[owner]) ; p_worker_cap -> Calculate the computacional capacity for each worker; dist_bruta -> Vector with the chunk numbers for each group in order to find the lower te_exec; prev_exec -> Runtime prediction for each worker.; temp_corr -> Calculates runtime deviation; task_exec -> Runtime for each task;

Min_Max Algorithm: It is a brute-force adjustment with a combinatorial optimization to find a chunk distribution with the smaller runtime possible. max_exec_total –> Finds the higher runtime prediction, reduces one unit from chunk number and recalculates again prev_exe, temp_corr and soma_dist. max_prev_exec–> Finds the higher runtime prediction for each worker.
